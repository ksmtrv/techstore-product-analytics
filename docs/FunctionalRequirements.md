# Функциональные требования

## Пользователи

Система должна хранить информацию о пользователях:

- ФИО;
- электронную почту;
- пол;
- дату рождения;
- город;
- дату регистрации.

---

## Категории товаров

Система должна хранить категории товаров.

Каждый товар принадлежит только одной категории.

---

## Товары

Для каждого товара необходимо хранить:

- название;
- бренд;
- цену;
- категорию.

---

## Заказы

Для каждого заказа хранится:

- пользователь;
- дата заказа;
- статус;
- способ оплаты;
- итоговая стоимость.

---

## Состав заказа

Каждый заказ состоит из одной или нескольких товарных позиций.

Для каждой позиции хранится:

- товар;
- количество;
- цена на момент покупки.

---

## Пользовательские сессии

Необходимо хранить:

- дату посещения;
- устройство;
- источник перехода;
- длительность сессии.
